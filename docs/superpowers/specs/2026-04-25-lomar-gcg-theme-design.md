# LoMar GCG — WordPress Theme Design Spec

**Date:** 2026-04-25
**Project:** lomar-gcg
**Status:** Approved

---

## Overview

Custom WordPress theme for LoMar GCG, a full-service landscape contractor in Northern Virginia. Direct port of the visual prototype in `design/` into a classic PHP WordPress theme. English only. Client manages portfolio content; everything else is hardcoded in templates.

---

| Token | Value | Use |
|-------|-------|-----|
| `--bg` | `oklch(0.978 0.005 88)` | Crema parchment — page background |
| `--bg-2` | `oklch(0.948 0.010 86)` | Section alternates |
| `--bg-3` | `oklch(0.912 0.014 84)` | Deeper sections |
| `--ink` | `oklch(0.15 0.04 50)` | Near-black warm — headings |
| `--ink-2` | `oklch(0.38 0.040 52)` | Body text |
| `--ink-3` | `oklch(0.56 0.028 56)` | Muted labels |

| `--accent` | `oklch(0.52 0.20 132)` | Forest green — CTAs, highlights |
| `--accent-ink` | `oklch(0.978 0.005 88)` | Text on accent backgrounds |

**Typography**

| Role | Font | Usage |
|------|------|-------|
| Display | Fraunces (italic, 400–600) | Hero headline, section titles |
| Body | Inter (400, 500) | Paragraphs, nav, buttons |
| Technical | JetBrains Mono | Eyebrows, labels, stats, sheet refs |

**Architectural aesthetic** — subtle blueprint/technical layer:
- Very faint grid on hero (`48px × 48px`, ~55% opacity)
- Dot grid on process section (`24px`, ~60% opacity)
- Corner marks on featured images (10px, 1px border, 50% opacity)
- Monospace labels and sheet reference annotations
- GPS coordinates in footer (`38°45'N 77°28'W`)

## Architecture

**Approach:** Classic PHP templates, direct port from `design/index.html`. No build tools. CSS custom properties + native `@import`. Vanilla JS. No jQuery.

**WordPress version:** 6.4+
**PHP version:** 8.2+
**Bedrock structure:** `web/app/themes/lomar-gcg/`

## Portfolio CPT

**Post type:** `project`
**Slug:** `/portfolio/{slug}`
**No individual pages** — lightbox opens from grid, no `single-project.php` needed.

**ACF Fields:**

| Field | Type | Notes |
|-------|------|-------|
| `project_photos` | Gallery | Min 1. First image = grid thumbnail |
| `project_service` | Select | Paver Patios, Garden Design, Fire Pits, Retaining Walls, Maintenance, Other |
| `project_location` | Text | e.g. "McLean, VA" |

**Post title** = project name (standard `post_title`).

**Lightbox:** GLightbox — clic on grid card opens gallery of all project photos. Card shows thumbnail + title overlay on hover.

**Homepage preview:** WP_Query for 6 latest `project` posts, reuses `portfolio-grid.php` partial. "View All Work →" link to `/portfolio`.

## Contact Form

**Plugin:** Contact Form 7. For the standalone `/contact` page: CF7 shortcode placed in page content via the WordPress editor (standard WP workflow — no PHP change needed). For the homepage section: shortcode hardcoded in `template-parts/contact.php`.

Fields: Name, Email, Phone (optional), Service interest (select), Message, Submit.

Submissions go to `wp_mail()` — no additional SMTP plugin required initially. CF7 styles overridden via `cf7-overrides.css` to match theme design system.

## Animations & Scroll Effects

All implemented with `IntersectionObserver` API + CSS transitions. No GSAP, no AOS.

| Effect | Implementation |
|--------|----------------|
| Section fade-in | `IntersectionObserver` triggers `.is-visible` class → `opacity + translateY` CSS transition |
| Nav shrink on scroll | `scroll` event listener → adds `.scrolled` class → CSS transitions padding + box-shadow |
| Hero parallax | `scroll` event → `background-position-y` adjustment. Disabled on mobile via media query |
| Smooth scroll | CSS `scroll-behavior: smooth` on `html` |
| Portfolio card hover | CSS `transition: opacity` on overlay. No JS |
| Services marquee | CSS `animation: marquee linear infinite`. No JS |
| GLightbox transitions | Built into GLightbox library |

## SEO & Performance

- `add_theme_support('title-tag')` — WordPress manages `<title>`
- All images as WebP, `loading="lazy"` on below-fold images
- GLightbox and any other per-page scripts loaded conditionally via `wp_enqueue_script` with template checks
- Cache busting via `wp_get_theme()->get('Version')` as asset version parameter
- Favicon: `favicon.ico` + `apple-touch-icon.png` via `wp_head`

**SEO Plugin: RankMath**

RankMath (free tier) — recommended over Yoast because the free tier includes LocalBusiness schema, 404 monitor, and redirections, all critical for a local service business.

Configuration on install:
- Business type: `LocalBusiness → LandscapeArchitect`
- Business name: LoMar GCG
- Address + phone + service area: Northern Virginia / DMV
- Opening hours
- Enable XML sitemap — submit to Google Search Console
- Enable Open Graph + Twitter Card meta (social sharing previews)
- Enable breadcrumbs (registered via `rank_math_breadcrumb`)

**Schema markup** — RankMath outputs `LocalBusiness` JSON-LD automatically. No manual schema needed in theme templates.

**Google Site Kit plugin** — connects Google Analytics 4 + Google Search Console directly in the WP dashboard. Client can see traffic data without leaving WordPress.

Configuration:
- GA4 property linked to site
- Search Console verified via Site Kit
- No manual `gtag` script in theme — Site Kit handles injection

**Object Cache (Hostinger Redis)**

Hostinger's hosting is optimized for Redis object cache. Setup:
- Plugin: `redis-cache` (Redis Object Cache by Till Krüss) — places drop-in at `web/app/object-cache.php`
- Add to `.env`: `REDIS_HOST=127.0.0.1` and `REDIS_PORT=6379` (Hostinger defaults)
- Add to `config/application.php`: `Config::define('WP_REDIS_HOST', env('REDIS_HOST'));`
- In Bedrock, the drop-in path is `web/app/object-cache.php` — the plugin handles this automatically
- Local dev: Redis optional. `WP_REDIS_DISABLED=true` in local `.env` to skip without errors
